The Big Picture: DDL vs DML

A Database Management System (DBMS) is the software that lets you define, create, maintain, and control access to a database. When you interact with a DBMS, you use a language called SQL (Structured Query Language). SQL has two main halves, and this week covers both of them.

The first half is the Data Definition Language (DDL). This is the "building" side of SQL. You use DDL to create your tables, set up the rules they must follow, change their structure, or remove them entirely. Think of DDL as the architect's blueprint: it defines what your database looks like before any data goes in.

The second half is the Data Manipulation Language (DML). This is the "doing" side. Once your tables exist, you use DML to put data in, pull data out, change data, or delete data. The most important DML command is SELECT, which is how you ask questions of your database.

Creating Tables with CREATE TABLE

Before you can store any data, you need to create a table to hold it. The CREATE TABLE statement tells the database: "Make me a new table with these columns, and each column stores this type of data."

The basic pattern looks like this:

CREATE TABLE TableName (
    column1    data_type,
    column2    data_type,
    column3    data_type
);

Every column needs a data type, which tells the database what kind of value it will hold. Here are the common ones you will use.

Data TypeWhat It StoresExample
NCHAR(size)A fixed-length text string. Always exactly that many characters long.NCHAR(4) stores exactly 4 characters, like 'D010'
NVARCHAR(size)A variable-length text string. Can be shorter than the maximum.NVARCHAR(20) stores up to 20 characters, like 'Smith'
INTA whole number (no decimal places).7, 12, 0
DECIMALA number that can have decimal places.15.75, 100.0
DATETIMEA date, and optionally a time.'2023/06/12' or '2023/06/12 12:30:00'
MONEYA currency value.450.00
BITA yes/no value (0 or 1).0 or 1
Example: Here is a simple table to store driver information. Each driver has an ID (fixed at 4 characters), a title, a name, and an address. 
CREATE TABLE Driver (
    driverid    NCHAR(4),
    title       NVARCHAR(4),
    name        NVARCHAR(20),
    address     NVARCHAR(40)
);
NCHAR vs NVARCHAR: Use NCHAR when the value is always the same length, like an ID code that is always 4 characters. Use NVARCHAR when values vary in length, like names or addresses. The "N" at the start means these support international (Unicode) characters.

Constraints: Rules Your Data Must Follow

When you create a table, you can also add constraints. These are rules that the database will enforce automatically. If someone tries to insert data that breaks a constraint, the database will refuse it. This is how the database protects itself from bad data.

There are several kinds of constraints, and they fall into three categories you have already met in earlier weeks: entity integrity, referential integrity, and domain integrity.

Figure 2: The main types of SQL constraints and what they do.

Here is a fuller example showing all of these in action. This creates a Deliver table with a primary key, two foreign keys, NOT NULL requirements, and CASCADE rules.

CREATE TABLE Deliver (
    delivery_id   NCHAR(7)      NOT NULL,
    driver_id     NCHAR(4)      NOT NULL,
    customer_id   NCHAR(6)      NOT NULL,
    vehicle       NCHAR(10),
    del_address   NVARCHAR(30)  NOT NULL,
    charge        MONEY,

    CONSTRAINT pk_Deliver PRIMARY KEY (delivery_id),

    CONSTRAINT fk_Deliver_Driver
        FOREIGN KEY (driver_id) REFERENCES Driver(driver_id)
        ON DELETE CASCADE ON UPDATE CASCADE,

    CONSTRAINT fk_Deliver_Customer
        FOREIGN KEY (customer_id) REFERENCES Customer(customer_id)
        ON DELETE CASCADE ON UPDATE CASCADE
);

The ON DELETE CASCADE part means: if a driver is deleted from the Driver table, automatically delete all their deliveries too. Without this, you would end up with delivery records pointing to a driver that no longer exists.

CHECK constraints let you set rules about what values are allowed in a column. For instance, you can say that a driver's points must be between 0 and 12, or that the title must be one of a specific set of values.

CREATE TABLE Driver (
    driver_id   NCHAR(4)       NOT NULL,
    firstname   NVARCHAR(30)   NOT NULL,
    lastname    NVARCHAR(30)   NOT NULL,
    points      INT            NOT NULL,

    CONSTRAINT pk_Driver PRIMARY KEY (driver_id),
    CONSTRAINT ck_driverid CHECK(driver_id LIKE 'D[0-9][0-9][0-9]'),
    CONSTRAINT ck_points CHECK(points >= 0 AND points < 12)
);
Watch out: Constraints have names (like pk_Driver or ck_points). These names must be unique within the database. Giving them clear, descriptive names makes it much easier to manage them later, for example when you need to drop or modify a specific constraint.

Altering and Dropping Tables

Once a table exists, you might need to change its structure. Perhaps you realise you need an extra column, or you want to add a constraint you forgot. The ALTER TABLE statement handles this.

The most common use is adding a new column to an existing table.

ALTER TABLE Payslip
ADD ni_num VARCHAR(10);

This adds a new column called ni_num to the Payslip table. Any existing rows will have NULL in this new column until you fill them in.

You can also use ALTER TABLE to add, switch off, or remove constraints after the table has been created.

-- Add a constraint to an existing table
ALTER TABLE Driver
ADD CONSTRAINT CK_points CHECK(points >= 0 AND points < 20);

-- Temporarily switch off a constraint
ALTER TABLE Driver NOCHECK CONSTRAINT CK_points;

-- Permanently remove a constraint
ALTER TABLE Driver DROP CONSTRAINT CK_points;

To remove an entire table from the database, you use DROP TABLE.

-- Remove a single table
DROP TABLE Payslip;

-- Remove a table and anything that depends on it
DROP TABLE Driver CASCADE;
Critical: DROP vs DELETE. These two commands do very different things. DROP TABLE destroys the entire table, its structure, its data, its constraints, everything. It is gone. DELETE (covered later) removes rows of data from a table but the table itself stays in place, ready to receive new data. If you just want to empty a table, use DELETE. If you want to remove the table from the database entirely, use DROP TABLE.

DROP TABLE Driver;

Destroys the Driver table completely. Structure, data, constraints, all gone.

DELETE FROM Driver;

Removes all the rows from Driver, but the table structure stays. You can insert new rows later.

Watch out: If another table has a foreign key pointing to the table you want to drop, you must use CASCADE. Otherwise the database will refuse, because dropping the table would leave orphaned foreign key references.

Adding Data with INSERT

Once your table exists, you add data to it one row at a time using the INSERT INTO statement. There are two ways to do this.

Method 1: Supply values for every column. If you are filling in all columns, you just list the values in the same order the columns were defined.

INSERT INTO Drivers
VALUES ('D060', 'Betty', 'Inverness', 6);
Figure 3: INSERT adds a new row to the bottom of the table.

Method 2: Supply values for only some columns. If you do not have values for every column, list the column names you are filling in. Any columns you leave out will be set to NULL (meaning "no value").

INSERT INTO Drivers (driverid, name)
VALUES ('D070', 'Jeannie');

In this case, the address and points columns for Jeannie will both contain NULL.

Watch out: If a column has a NOT NULL constraint, you cannot leave it out. The database will reject the insert because that column is required to have a value.

Method 3: INSERT with SELECT. You can also copy rows from one table into another by combining INSERT INTO with a SELECT query. The target table must already exist.

-- Copy all Dundee drivers into an archive table
INSERT INTO Driver_Archive
  SELECT *
  FROM Drivers
  WHERE address = 'Dundee';

This finds every driver with a Dundee address and copies their full row into the Driver_Archive table. You can also copy only specific columns, but any columns you do not fill will be set to NULL in the archive.

Retrieving Data with SELECT

The SELECT statement is the most important command in SQL. It is how you ask questions of your database and get answers back. The result of a SELECT query is always a temporary table, which means you get back a set of rows and columns, but this result is not stored permanently.

An important thing to understand is that SELECT only reads data. It never changes the data in your tables. It is a safe, read-only operation.

The basic structure of a SELECT query has three parts.

Figure 4: The three-part structure of a SELECT query. The WHERE clause is optional.

Here are the three basic patterns you need to know.

Select specific columns: List the column names you want, separated by commas.

SELECT driverid, points
FROM Drivers;

This gives you just those two columns for every row in the table.

Select all columns: Use the * wildcard to get everything.

SELECT *
FROM Drivers;

Filter with WHERE: Add a condition to get only the rows that match.

SELECT driverid
FROM Drivers
WHERE address = 'Dundee';

This gives you only the driver IDs of drivers whose address is Dundee.

Figure 5: A SELECT with WHERE filters rows (green) and picks only the requested columns.
Good to know: Text values in SQL must always be wrapped in single quotes, like 'Dundee'. Numbers do not need quotes. Getting this wrong is a very common source of errors.

Extending SELECT: LIKE, IN, ORDER BY, and Complex Conditions

The basic SELECT ... FROM ... WHERE pattern is powerful, but SQL gives you several extra tools to make your queries more precise. This section covers the most important ones.

LIKE: Pattern matching. Sometimes you do not know the exact value you are looking for, but you know what it starts with, ends with, or contains. The LIKE clause lets you use the % symbol as a wildcard, meaning "any characters here."

-- Find drivers whose address starts with 'Du'
SELECT name, address
FROM Drivers
WHERE address LIKE 'Du%';

This matches "Dundee", "Dublin", "Dumfries", or any address starting with "Du". The % stands in for anything that follows.

'Du%'

Starts with "Du", then anything. Matches Dundee, Dublin.

'%ee'

Ends with "ee". Would match Dundee, but not Dublin.

'%und%'

Contains "und" anywhere. Matches Dundee.

IN: Checking a list of values. If you want to match against several specific values, the IN operator is cleaner than writing lots of OR conditions.

-- Drivers from Dundee or Aberdeen
SELECT name
FROM Drivers
WHERE address IN ('Dundee', 'Aberdeen');

-- Drivers NOT from Dundee or Aberdeen
SELECT name
FROM Drivers
WHERE address NOT IN ('Dundee', 'Aberdeen');

ORDER BY: Sorting results. By default, SQL does not guarantee any particular order for your results. If you want them sorted, you must say so. The default sort is ascending (A to Z, or smallest to largest). Add DESC to reverse it.

-- Alphabetical order (ascending is the default)
SELECT name
FROM Drivers
WHERE address IN ('Dundee', 'Aberdeen')
ORDER BY name;

-- Reverse alphabetical order
SELECT name
FROM Drivers
WHERE address IN ('Dundee', 'Aberdeen')
ORDER BY name DESC;

Combining conditions with AND and OR. You can build complex filters by joining conditions together. AND means both conditions must be true. OR means at least one must be true. You can also use comparison operators like >=, <=, and <> (which means "not equal to").

-- Points between 4 and 15, and NOT from Dundee
SELECT name
FROM Drivers
WHERE points >= 4
  AND points <= 15
  AND (address <> 'Dundee');
-- Address starts with 'D' OR starts with 'A'
SELECT name
FROM Drivers
WHERE (address LIKE 'D%')
   OR (address LIKE 'A%');
Watch out: When mixing AND and OR in the same query, use brackets to make the grouping clear. Without brackets, the database applies AND before OR, which can give you unexpected results. If in doubt, add brackets.

Aggregate Functions and Calculated Columns

So far, every query has returned individual rows from the table. But sometimes you want a summary, for example "how many drivers are there?" or "what is the average number of points?" SQL provides five built-in aggregate functions (also called column functions) for exactly this.

FunctionWhat It DoesExample
COUNT(column)Counts the number of values in a column.COUNT(driverid) returns 5
SUM(column)Adds up all the values in a column.SUM(points) returns 28
AVG(column)Calculates the average of all values.AVG(points) returns 5.6
MAX(column)Finds the largest value.MAX(points) returns 12
MIN(column)Finds the smallest value.MIN(points) returns 2

You can use several aggregate functions in a single query, and you can even do arithmetic with them.

SELECT COUNT(driverid), SUM(points), AVG(points),
       MAX(points), MIN(points),
       MAX(points) - MIN(points)
FROM Drivers;

This returns a single row with six values: the count (5), the total (28), the average (5.6), the maximum (12), the minimum (2), and the range (10).

Good to know: When you use aggregate functions, the result is always a single summary row, not a row per driver. The database crunches all the rows together into one answer. You will learn about GROUP BY later, which lets you get summaries per group (for example, average points per city).

Calculated columns with AS. You can also do maths within a SELECT and give the result a name using the AS keyword. This does not change any data in the table. It just creates a temporary extra column in your results.

SELECT item, qty * price AS TotalCost
FROM Items_ordered;

This multiplies the quantity by the price for each row and shows it in a column called "TotalCost". The available mathematical operators are + (add), - (subtract), * (multiply), / (divide), and % (modulo, which gives the remainder after division).

Example: Using the Items_ordered table, if Shoes have a quantity of 2 and a price of 20, the TotalCost column would show 40. The original qty and price columns are unchanged.

Exercise 1: DDL or DML?

You want to add a new column called "email" to the Student table. Which category of SQL does this belong to?

DML (Data Manipulation Language)
Adding a column changes the structure of a table, so this is DDL. You would use ALTER TABLE Student ADD email NVARCHAR(50);

DDL (Data Definition Language)
DML (Data Manipulation Language)
Retrieving data is working with the data itself, not the table structure, so this is DML. You would use a SELECT query with a WHERE clause.

You want to remove the Payslip table from the database entirely. Which category is this?

DDL (Data Definition Language)
DML (Data Manipulation Language)
Removing a table destroys its structure, so this is DDL. You would use DROP TABLE Payslip;

Exercise 2: Choosing the Right Data Type

You need a column to store a driver's licence number, which is always exactly 16 characters. Which data type is best?

NCHAR(16)
NVARCHAR(16)
INT
Because the value is always exactly 16 characters, NCHAR(16) is the right choice. NVARCHAR is for values that vary in length. INT is for whole numbers, not text.

You need a column to store people's home addresses, which vary in length. Which data type is best?

NCHAR(50)
NVARCHAR(50)
DATETIME
Addresses vary in length (some are short, some are long), so NVARCHAR(50) is the right choice. NCHAR(50) would waste space by padding every address to exactly 50 characters.

Exercise 3: DROP vs DELETE

You want to remove all the rows from the Orders table, but keep the table itself so you can add new orders next month. What should you use?

DROP TABLE Orders;
DELETE FROM Orders;
ALTER TABLE Orders;
DELETE removes the data (rows) but keeps the table structure intact so you can use it again. DROP TABLE would destroy the entire table, and you would have to recreate it from scratch.

A table called "TempData" is no longer needed at all. You want to remove it and everything about it from the database. What should you use?

DROP TABLE TempData;
DELETE FROM TempData;
DROP TABLE removes the table entirely, including its structure, constraints, and data. Since you do not need the table at all any more, this is the right choice.

Exercise 4: Reading SELECT Queries

The SELECT name part means we only get the name column back. The WHERE address LIKE 'A%' filters to rows where the address starts with the letter 'A'. So the result is the names of drivers whose address begins with 'A' (for example, Aberdeen).

What does this query return?

All drivers sorted by points from lowest to highest

The * means all columns. The WHERE points >= 5 filters to only drivers with 5 or more points. The ORDER BY points DESC sorts the results in descending order, so the highest points appear first.

Exercise 5: Writing the Right Query

You want to find out how many drivers are in the Drivers table. Which query would you use?

SELECT * FROM Drivers;
SELECT COUNT(driverid) FROM Drivers;
SELECT SUM(driverid) FROM Drivers;
COUNT counts the number of values in a column, which gives you the number of drivers. SELECT * would show you all the data but would not give you a count. SUM adds up numeric values, and driver IDs are text, so that would not work.

You want to list the names of drivers who are NOT from Perth or Stirling. Which WHERE clause is correct?

WHERE address IN ('Perth', 'Stirling')
WHERE address NOT IN ('Perth', 'Stirling')
WHERE address LIKE 'Perth' OR address LIKE 'Stirling'
NOT IN excludes the listed values. The first option (IN) would do the opposite, returning only Perth and Stirling drivers. The third option uses LIKE without a wildcard, so it would find matches rather than exclude them.

A CHECK constraint validates that values fall within a specific range or match a specific pattern. You would write something like CHECK(age >= 0 AND age <= 150). PRIMARY KEY is for unique row identifiers, FOREIGN KEY is for linking tables, and NOT NULL just ensures a value is present.
